1.	
a.	Pengertian single responsibility ?
Setiap kelas harus memiliki satu tanggung jawab, dan tanggung jawab itu harus sepenuhnya dirangkum oleh kelas ,dan setiap class tidak boleh lebih dari 1 tanggung jawab

Keuntungan 

-	Dapat memodifikasi code dengan mudah dan code menjadi lebih rapih
-	Menghindari konflik ketika merge request
-	Untuk mengurangi kompleksitas saat terjadi perubahan
-	Menurunkan tingkat kebergantungan antar class. Semakin tinggi kebergantungan semakin susah untuk dimodifikasi
b.	Apa itu prinsip open/close ?
Adalah suatu program akan bersifat tertutup untuk modifikasi atau perubahan pada sebuah program seperti perubahan interface ,extension, namun terbuka untuk penambahan. Prinsip ini sangat penting bagi programmer pada sebuah program yang akan dipublikasikan ,karena agar tidak dapat dirubah seenaknya oleh programmer lain dan menjadikan terjaga dari perubahan program
Keuntungan 
-	Kode menjadi lebih sederhana
-	Untuk testing menjadi lebih mudah 
-	Menghindari terjadinya kesalahan
